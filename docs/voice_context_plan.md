## שלב 1 — תיקון סנכרון הזדהות

### T1.1 — frontend: שידור auth על WS פתוח
ב-`SadanChat.jsx`: האזן ל-`sadan:authenticated` (שכבר משוגר מ-`Login.jsx` בהזדהות ידנית).
אם ה-WS פתוח — שלח `{type:"auth_context", authenticated:true, manual:true}` מיד.
(המנגנון הקיים ב-`ws.onopen` נשאר — הוא מכסה חיבור חדש אחרי הזדהות.)

### T1.2 — backend: קליטת auth באמצע session
ב-`gemini_live_pipeline.py`, בטיפול ב-`auth_context` בתוך `_send_audio`:
בנוסף ל-`self._authenticated=True`, שלח ל-Gemini הודעת מערכת:
`[מידע מערכת: המשתמש השלים הזדהות ידנית במסך. מצב ההזדהות הסתיים. עבור למצב עבודה מלא — כל הכלים זמינים. אל תבקש מספר אישי.]`

### T1.3 — ניסוח מחדש של ה-system prompt
מצב ההזדהות מוגדר כ**מצב זמני** ("עד שתקבל אישור הזדהות — מהכלי או מהודעת מערכת"),
לא כזהות קבועה. ההנחיות הקיימות נשארות, רק המסגור משתנה.

## שלב 2 — Context Registry: סדן רואה את המסך

### T2.1 — `frontend/src/services/sadanContext.js`
Singleton קטן:
```js
setScreen(screen, state)   // מסך חדש + מצב התחלתי
patch(partial)             // עדכון מצב בתוך המסך (טאב, שדה, בחירה)
get()                      // { screen, state }
subscribe(cb)              // SadanChat מאזין
```
כל קריאה משגרת עדכון (debounced 400ms) למנויים.

### T2.2 — SadanChat: שידור context_update
מנוי על sadanContext; כשיש שינוי וה-WS פתוח — שלח
`{type:"context_update", screen, state}` (קומפקטי, רק שדות שאינם ריקים).

### T2.3 — backend: קליטת context_update
ב-`_send_audio`: הודעת מערכת ל-Gemini:
`[מידע מערכת: מסך=<שם> | <state כתיאור קצר>]` — משפט אחד, לא JSON גולמי.

### T2.4 — SCREEN_GUIDE בצד השרת
מילון סטטי ב-`gemini_live_pipeline.py`: לכל מסך 5–10 שורות —
מה המשתמש רואה, אילו פעולות זמינות, אילו כלים רלוונטיים, ניסוח הדרכה.
נשלח ל-Gemini פעם אחת בכל מעבר מסך (יחד עם ה-screen_change הקיים).
מסכים: field-selection, area, calendar, questionnaire, plans, exercise, quiz, approvals, simulation.

### T2.5 — חיווט המסכים (דיווח state)
כל מסך מדווח בכמה שורות (useEffect):
| מסך | מה מדווח |
|---|---|
| Questionnaire | ערכי שדות שמולאו (כשירות, תחמושת, תאריך, שת"פ...) |
| Plans | איזה מתווה נבחר / אף אחד |
| Exercise | הטאב הפעיל ב-sidebar |
| Quiz | מספר שאלה נוכחית, כמה נענו, ציון אם הוגש |
| Approvals | סטטוסים (כמה אושרו/ממתינים), שיחה פעילה |
| Area | פאנל פתוח (שטח נבחר), 2D/3D |
| AreaCalendar | שטח מסונן, טווח נבחר |
| Simulation | שלב נוכחי, רץ/מושהה |
| FieldSelection | (סטטי — רק שם מסך) |
