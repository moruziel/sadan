# תוכנית מערכת בדיקות E2E — סדן

> **מטרה:** לוודא אוטומטית, לפני כל חזרה ובכל לילה, שהדמו שלם: הזרימה עובדת, ה-UI נקי מחפיפות, והקיר מציג את מה שצריך.
> **כלי:** Playwright (Chromium). **יעד ברירת מחדל:** השרת בענן (מה שרץ בפגישה). ניתן להריץ גם מול סביבה מקומית.
> **מיקום:** `tests/e2e/`

## ארכיטקטורה
- **שני "סימולטורים" במקביל:** context טלפון (375×812) שמריץ את הדמו + context קיר (1600×900) שמציג את `/wall`. הבדיקה מנווטת בטלפון ומוודאת שהקיר הגיב.
- **הפעלת פעולות:** דרך אותם אירועים שהקול ופאנל הדמו משתמשים בהם (`sadan:navigate`, `fillField`, `sadan:action`) — בדיקה של התנהגות אמיתית בלי תלות במיקרופון.
- **אימות בסיסי:** דרך `httpCredentials` (basic auth של האתר).
- **גבול מכוון:** הבדיקות לא לוחצות "התקשר" (שיחה אמיתית!) — תמלול שיחה נבדק בהזרקת אירועים לערוץ הקיר. שיחה אמיתית = `test-remote.ps1`, ידני לפני חזרות.

## הבדיקות

| קובץ | מה בודק | קריטריון |
|---|---|---|
| `demo-flow.spec.js` | **התרחיש המלא בשני מסכים:** כניסה ידנית → בחירת שטח → מפה → שאלון (מילוי בקול-מדומה) → מתווים → תיק (כולל טאבי חובה) → בוחן → אישורים. בכל תחנה: המסך נטען, הקיר מציג את התחנה, צילום מסך | כל תחנה עוברת; הקיר מסונכרן; "סדן מנתחת מתווים" + ציון 92 מופיעים |
| `wall-events.spec.js` | ערוץ הקיר ישירות: אירוע context, תמלול שיחה (שני צדדים), תמלול צ'אט | כל אירוע מופיע על הקיר תוך <2 שניות |
| `overlap.spec.js` | לכל מסך, בטלפון ובדסקטופ: אף אלמנט אינטראקטיבי לא חורג מהמסך ולא חופף לאחר; אפס שגיאות console | 0 חפיפות, 0 חריגות, 0 שגיאות |

## הרצה
- `run-e2e.ps1` — ריצה ידנית (פרמטר `-Headed` להצגה חיה של שני החלונות על המסך)
- דוח HTML אוטומטי עם צילומים וסרטון לכל כשל
- **תזמון:** משימת Windows יומית (06:00) מול הענן, headless; הדוח נשמר ב-`tests/e2e/reports`

## מה נשאר ידני (במכוון)
איכות הקול והדיבור של סדן — חזרות גנרליות. שיחת טלפון/וואטסאפ אמיתיים — `test-remote.ps1`.
