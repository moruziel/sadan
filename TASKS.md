# שלב A — תשתית קול

## TASK-001: הקמת ElevenLabs ⬜

**מטרה:** יכולת להפוך טקסט עברי לדיבור טבעי.

**פעולות:**
1. מור יספק API key של ElevenLabs (הוא יצור חשבון וישלם)
2. צור קובץ `backend/services/tts_service.py`:
   ```python
   class TTSService:
       def text_to_speech(self, text: str, voice_id: str) -> bytes:
           """מקבל טקסט בעברית, מחזיר MP3"""
   ```
3. בחר קול עברי מ-ElevenLabs — **גברי, סמכותי, לא רובוטי.** אם אין קול עברי מובנה — צור voice clone או השתמש ב-multilingual v2.
4. כתוב בדיקה שמייצרת MP3 מהמשפט: "שלום, כאן מערכת סדן. אני פונה אליך בנוגע לתיאום תרגיל."
5. שמור את הMP3 ב-`demo_assets/voice_samples/test_tts.mp3`
6. **הצגה למור:** השמע את הקובץ. מור יחליט אם הקול מספיק טוב.

**Dependencies:** `pip install elevenlabs --break-system-packages`

**מה אסור:** לא להתקדם בלי שמור שמע ואישר את הקול.

**שאלה למור אם נתקע:** "אין קול עברי טוב מספיק. אפשרויות: (1) multilingual v2 (2) voice clone מהקלטה (3) לשנות ל-Google Cloud TTS. מה מעדיף?"

## TASK-002: הקמת Whisper (STT) ⬜

**תלוי ב:** TASK-001 מאושר

**מטרה:** יכולת לתמלל דיבור עברי לטקסט.

**פעולות:**
1. צור `backend/services/stt_service.py`:
   ```python
   class STTService:
       def speech_to_text(self, audio: bytes) -> str:
           """מקבל אודיו, מחזיר טקסט בעברית"""
   ```
2. **שתי אפשרויות — בדוק שתיהן ובחר:**
   - **OpenAI Whisper API** — מהיר, מדויק, עלות קטנה ($0.006/דקה). דורש API key של OpenAI.
   - **Whisper מקומי** (`openai-whisper` Python package) — חינם, אבל איטי יותר, דורש GPU לביצועים טובים.
3. **המלצה:** Whisper API (ענן) — אמין יותר לדמו. אם מור מעדיף מקומי — שאל.
4. בדיקה: הקלט משפט בעברית מהמיקרופון → תמלל → הצג תוצאה.
5. **בדוק דיוק בעברית צבאית:** תמלל "אני צריך תרגיל מחלקה רטוב בשטח בנוי" — אם יוצא שטויות, דווח למור.

**Dependencies:** `pip install openai --break-system-packages` (אם API) או `pip install openai-whisper --break-system-packages` (אם מקומי)

## TASK-003: הקמת Twilio Voice ⬜

**תלוי ב:** TASK-001 מאושר (TTS עובד)

**מטרה:** יכולת ליזום שיחת טלפון, להשמיע הודעה, ולהקליט תגובה.

**פעולות:**
1. מור יספק Twilio account + מספר טלפון. **שאל אותו:**
   - "האם יש לך חשבון Twilio? אם לא, צריך ליצור ב-twilio.com"
   - "צריך מספר טלפון — ישראלי (אם זמין) או אמריקאי. מה מעדיף?"
   - "מה המספר של רז לבדיקות?"
2. צור `demo_assets/call_server/server.js`:
   ```javascript
   // Express server on port 3002
   // POST /call — יוזם שיחת טלפון
   //   body: { to: "+972...", script_id: "rtg_request" }
   //   flow:
   //     1. Twilio calls the number
   //     2. When answered → play pre-recorded TTS message (ElevenLabs MP3)
   //     3. Record the response
   //     4. Send recording to Whisper STT
   //     5. Claude API processes the text
   //     6. Generate TTS response → play back
   //     7. Repeat until conversation ends
   //     8. POST result to backend
   // GET /call-status/:id — סטטוס שיחה
   ```
3. **לדמו — מצב בטוח:** הכן **3 scripts מוקלטים מראש** (ElevenLabs):
   - `rtg_request.mp3` — "שלום, כאן מערכת סדן מגדוד 51. אני פונה בנוגע לתיאום תרגיל מחלקה בשטח 309ה בטונדות, בתאריך 5 במאי. תרגיל רטוב, תחמוש 5.56 וחבלה, דימוי אויב. האם תוכל לאשר?"
   - `matam_gonogo.mp3` — "שלום, כאן מערכת סדן. כל התיאומים לתרגיל מחלקה בשטח 309ה אושרו. מבקשים אישור GO לכניסה לשטח מחר בשעה 06:00."
   - `helipad_request.mp3` — "שלום, כאן מערכת סדן מגדוד 51. מבקשים תיאום מסוק פינוי פצועים מדומה לתרגיל בשטח 309ה, ב-5 במאי, חלון 10:00 עד 12:00. נקודת הנחתה בנ.צ. 33.044, 35.650."
4. **Webhook:** Twilio צריך URL חיצוני לשלוח webhook. פתרונות:
   - `ngrok` — tunnel מהמחשב של מור
   - או: Twilio Studio (flow builder, לא צריך webhook)
   - **שאל את מור איזה גישה מעדיף**
5. בדיקה: התקשר למספר של מור, השמע הודעת טסט, הקלט תגובה, תמלל.

**Dependencies:**
```
npm install twilio express cors
npm install -g ngrok   (אם נבחר)
```

**⚠️ סיכון:** Twilio + webhook + ngrok = 3 שכבות שיכולות להישבר. **חלופה אם נתקע:** WhatsApp voice notes (הודעות קוליות) דרך whatsapp-web.js — פחות מרשים אבל יותר יציב. **אל תעבור לחלופה בלי אישור מור.**

## TASK-004: Voice Pipeline — חיבור הכל ⬜

**תלוי ב:** TASK-001 + TASK-002 + TASK-003 מאושרים

**מטרה:** pipeline שלם: טקסט → TTS → שיחה → STT → AI → TTS → שיחה.

**פעולות:**
1. צור `backend/services/voice_pipeline.py`:
   ```python
   class VoicePipeline:
       def __init__(self, tts: TTSService, stt: STTService):
           ...
       
       def create_call_script(self, template: str, data: dict) -> str:
           """ממלא template עם נתוני התרגיל"""
       
       def process_response(self, transcription: str, context: dict) -> str:
           """שולח ל-Claude API, מקבל תגובה"""
       
       def handle_call(self, call_type: str, data: dict):
           """מנהל שיחה מלאה — מתקשר, מדבר, מקשיב, עונה"""
   ```
2. צור 3 templates לשיחות:
   - `rtg_call_template` — פנייה לפקח רטג, כולל כל פרטי התרגיל
   - `matam_call_template` — בקשת GO/NO-GO ממתא"ם
   - `heli_call_template` — תיאום מסוק פינוי עם חי"א
3. לכל template — **2 מצבים:**
   - `live` — AI אמיתי שמאלתר (Claude API)
   - `scripted` — הודעות מוקלטות + זיהוי מילת מפתח ("מאשר" / "דחוי")
4. **ברירת מחדל לדמו: `scripted`.** מור יחליט אם לעבור ל-`live`.
5. בדיקה מלאה: שיחה אמיתית למספר של מור עם script רטג. מור עונה "מאשר". המערכת מזהה ומעדכנת.

**עצירה:** הצג למור הקלטה של שיחה שהצליחה. **זה checkpoint קריטי** — אם השיחות לא עובדות, הדמו צריך plan B.
